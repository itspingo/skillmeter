SkillMeter API Usage Guide

Base URL: All API endpoints are relative to your application's base URL (e.g., http://localhost:8000/).

Authentication: Most endpoints are protected and require a Bearer Token. First, log in to get a token, then include it in the Authorization header for all subsequent requests: Authorization: Bearer {your_token}.

Authentication

POST/register

Register a new user.

Request Body: 
{
    "name": "John Doe",
    "email": "john.doe@example.com",
    "password": "password123",
    "password_confirmation": "password123",
    "user_type_id": 2 // Or appropriate type ID
}

POST/login

Log in to receive an authentication token.

Request Body: 
{
    "email": "john.doe@example.com",
    "password": "password123"
}
Success Response: 
{
    "token": "1|xxxxxxxxxxxxxxxxxxxxxxxxxxxxxx",
    "user": { ... }
}

GET/me

Get the authenticated user's details. (Requires Bearer Token)

POST/logout

Log out the current user. (Requires Bearer Token)

Tests

Manage skill assessment tests. All endpoints require authentication.

GET/tests

List all tests created by the authenticated user.

POST/tests

Create a new test.

Request Body: 
{
    "title": "Advanced PHP Test",
    "description": "A test to evaluate advanced PHP skills.",
    "duration_minutes": 60
}

GET/tests/{id}

Get details of a specific test.

PUT/tests/{id}

Update a test.

DELETE/tests/{id}

Delete a test.

POST/tests/{test_id}/questions

Add questions to a test.

Request Body: 
{
    "questions": [
        { "question_id": 1, "weight": 2 },
        { "question_id": 5, "weight": 1 }
    ]
}

POST/tests/{test_id}/invite

Invite candidates to take a test.

Request Body: 
{
    "emails": ["candidate1@example.com", "candidate2@example.com"],
    "expires_at": "2025-12-31 23:59:59"
}

GET/tests/invitation/{token}

Get test details via a public invitation token. (No Auth Required)

Questions

Manage questions for tests. All endpoints require authentication.

GET/questions

List all questions created by the authenticated user.

POST/questions

Create a new question.

Request Body: 
{
    "question_text": "What is the output of `echo 'a' . 'b';`?",
    "question_type_id": 1, // MCQ
    "question_category_id": 3, // PHP
    "difficulty_level_id": 2, // Intermediate
    "options": [
        { "option_text": "ab", "is_correct": true },
        { "option_text": "a b", "is_correct": false }
    ]
}

POST/questions/generate-ai

Submit a request to generate questions using AI.

Request Body: 
{
    "topic": "Laravel Eloquent",
    "difficulty": "Intermediate",
    "question_type": "MCQ_Single",
    "count": 5
}

Test Attempts

View and manage test attempts.

GET/attempts

List test attempts. Recruiters see attempts for their tests; candidates see their own. (Requires Auth)

GET/attempts/{id}

Get details of a specific test attempt. (Requires Auth)

POST/tests/invitation/{token}/start

Start a test using an invitation token. (No Auth Required)

POST/attempts/{attemptId}/submit

Submit a test attempt. (Requires Auth)

Request Body: 
{
    "responses": [
        { "question_id": 1, "selected_option_ids": [3] },
        { "question_id": 2, "answer_text": "This is a text answer." }
    ]
}

Cheating Detection

Endpoints for proctoring and cheating data analysis. (Requires Auth)

GET/cheating

List cheating incidents. Scoped to the user's role.

POST/cheating

Store a new cheating incident.

Request Body: 
{
    "test_id": 15,
    "event_type": "Multiple Faces Detected",
    "confidence": "0.95",
    "process_time": "2025-08-08 10:30:00"
}

GET/test/{testId}/summary

Get a summary of cheating incidents for a specific test.
